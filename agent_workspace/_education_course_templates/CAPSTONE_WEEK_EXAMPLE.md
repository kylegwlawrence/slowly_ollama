# Week 31: Capstone Proposal

> **Worked example of [`CAPSTONE_WEEK_TEMPLATE.md`](CAPSTONE_WEEK_TEMPLATE.md).** Subject: undergraduate physics capstone. Use it as a target shape when filling a capstone-week outline. Counts match the template (3 deliverables, 6 activities, 3 tools/methods, 3 reflection prompts). The Score column in the rubric is intentionally blank — the learner fills it after doing the work. ## Metadata

- **Capstone phase**: Proposal
- **Duration**: ~10–15 hours total this week
- **Prerequisites**: Units 5–9 content (the project's physics will draw on at least one of these). Familiarity with reading research papers helps but isn't required.

## Phase Overview

The Proposal phase converts a vague interest into a scoped project you can actually finish in two weeks of execution. By end of week you have a clear research question, a foundation of read literature, and the tooling stood up — so Week 32 begins on day one with execution, not setup.

## Deliverables

Concrete artifacts you should have by end of week.

- **Two-page project proposal**: one-sentence research question + three sub-questions, methodology (theoretical / simulation / data-analysis / experimental), expected outcomes, scope, two-week timeline, and a "what could go wrong" paragraph
- **Annotated literature foundation**: 100-word summaries of ≥5 relevant papers (contribution, method, your reaction), with full citations
- **Project repository**: a git repo (GitHub or local) with a README stating the question and one-paragraph methodology, a `notes/` folder, and a smoke-tested placeholder for the code or LaTeX you'll write in Week 32

## Activities

Task-oriented, not hour-by-hour. Work in whatever order suits the project.

- [ ] Pick a topic from Units 5–9 that genuinely interests you. Examples: a quantum-algorithm walkthrough, a Bell-test reproduction, a chaos-theory visualization, a Lagrangian-mechanics derivation for a non-trivial system, a statistical-mechanics phase-transition analysis
- [ ] Search [arXiv](https://arxiv.org/) and [Google Scholar](https://scholar.google.com/) for 5+ papers on the chosen topic. Bias toward review papers for breadth plus 1–2 primary papers for depth
- [ ] Write a 100-word summary of each paper covering contribution, method, and your reaction
- [ ] Draft the research question. Test it: can you state in one sentence what answering it would tell you? Can you list three sub-questions that together would resolve it?
- [ ] Write the 2-page proposal. Include question, why it matters, methods, expected outcomes, scope, two-week timeline, and what could go wrong
- [ ] Stand up the project repo (initial commit, README, `notes/` folder) and run one smoke-test per tool you'll need in Week 32 — confirm each works on your machine before execution week starts

## Tools & Methods

- **[arXiv](https://arxiv.org/) + [Google Scholar](https://scholar.google.com/)** — literature search and citation lookup this week; don't yet commit to a reference manager
- **[LaTeX](https://www.latex-project.org/) (Overleaf or local)** — drafting the 2-page proposal and the eventual writeup
- **Python (NumPy / Matplotlib) + git** — smoke-testing the toolchain and committing the repo skeleton so Week 32 starts in execution mode, not setup mode

## Self-Assessment Rubric

Self-grade this week's deliverables. Be honest — the rubric is for you.

| Criterion | What it means | Score (1–5) |
|-----------|---------------|-------------|
| **Rigor** | Is the research question specific enough to answer in two weeks? Are claims in the proposal sourced to specific papers? | |
| **Completeness** | All three deliverables present and finished — proposal, ≥5 paper summaries, repo skeleton? | |
| **Communication** | Will future-you (in 6 months) understand the proposal? Could someone else execute the methodology you wrote? | |
| **Technical depth** | Is the proposal engaging real Unit 5–9 physics, or just summarizing introductory material? | |

**Total**: __ / 20

If total < 12: pick the lowest-scoring criterion and spend extra time on it before starting Week 32. Usually that means the scope is too broad or the question is too vague.

## Reflection Prompts

Write briefly (3–5 sentences each) on 2–3 of these at week end:

- Which Unit 5–9 concept does this project most depend on, and how confident are you in that prerequisite knowledge?
- What's the single biggest risk to your project right now, and how will you mitigate it in Week 32?
- What would you cut from the scope if Week 32 ran short on time? Have an answer ready now, not under pressure later.
