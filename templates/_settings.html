{# `hx-on::after-request` does triple duty: on success it resets the form, hides the error region, and sets the icon to a green check; on any non-2xx it pipes the plain-text response body into the error region and sets the icon to a red ✗. HTMX still skips the target swap on non-2xx (good — the list stays intact) so the error region + icon are the only things the user sees change. `hx-on::before-request` clears the icon so a stale ✓/✗ from the previous submit doesn't linger while the new probe runs. The 4xx/5xx body is plain text (e.g. "Server name 'arxiv' already in use.") so assigning to `textContent` is the right escape boundary — never use innerHTML here. The icon DOES use innerHTML, but only with a static string literal we control. #}
