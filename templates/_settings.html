{# `hx-on::after-request` does double duty: on success it resets the form and hides the error region; on any non-2xx it copies the response body into the error region and unhides it. HTMX still skips the target swap on non-2xx (good — the list stays intact) so the error region is the only thing the user sees change. The 4xx body is plain text (e.g. "Server name 'arxiv' already in use.") so assigning to `textContent` is the right escape boundary — never use innerHTML here. #}
