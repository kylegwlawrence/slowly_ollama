{# Form lifecycle is handled by the `.rag-server-form` branches in static/app.js: - htmx:beforeRequest clears the icon so a stale ✓/✗ from the previous submit doesn't linger while the new probe runs. - htmx:afterRequest on success: reset the form, hide the error region, paint the green check_circle into #health-check-icon. - htmx:afterRequest on non-2xx: write the plain-text response body into the error region (textContent — never innerHTML — since the body is untrusted), unhide it, paint the red cancel glyph. HTMX skips the target swap on non-2xx, so the list stays intact and the user only sees the error + icon change. #}
