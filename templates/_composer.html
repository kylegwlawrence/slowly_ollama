{# Empty-state composer. Shown by `index_endpoint` when no chat is open, and by `new_chat_endpoint` (GET /new) when the user clicks the sidebar "+ New chat" link from inside an existing chat. Submitting the form creates a new conversation AND saves the first user message in a single round trip (see `create_chat_endpoint` in app/routes.py). The response replaces #main with the rendered chat panel (which includes the user's message and an SSE placeholder for the assistant's reply) and OOB-prepends the new sidebar row. The pill is the shared `.input-pill` block from style.css — same styling as the in-chat message form so both inputs feel identical. The model select sits in a toolbar below the pill (Claude-style), and auto-loads options from GET /models on insert. #}
